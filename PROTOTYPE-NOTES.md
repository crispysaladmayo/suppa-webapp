# Prototype notes (M1 hi-fi HTML)

1. **Brand:** Customer-facing product name is **Suppa** ([`suppa-brand-framework.md`](../suppa-brand-framework.md)). Landing uses the **Suppa** wordmark (Fraunces), positioning line from the brand framework, and PRD-aligned disclaimers (“Suppa shares…”). Screen `<title>` elements use the “— Suppa” suffix where applicable.
2. **0–5 month Today (milk mode):** See [`today-infant.html`](today-infant.html) + [`log-feed.html`](log-feed.html)—no solid macro chart, **Record a feed** CTA, **generic** recipe strip + **Browse all safe recipes**. Default demo remains [`today.html`](today.html) (**3–5 years**, macros + gap hints).
3. **Auth:** M1 is **email + password**, min **8 characters** (PRD FR-A1). **Forgot password** → [`forgot-password.html`](forgot-password.html) → demo “check email” state → demo link to [`reset-password.html`](reset-password.html?token=demo). Real email + tokens are engineering. **Today** greeting is **“Good [time], mama — [child]”** via [`app.js`](app.js) (`data-child-first` on `#today-greeting`).
4. **Recipe detail:** **Share** opens a bottom sheet (**Copy link**, **Print or PDF**, **Copy recipe text**, **Close**). **Revoke share link** appears only for **`?mine=1` (Yours)**—**Library** hides it (PRD §8.5 default). Revoke uses `confirm()` + toast (engineering: real token + API).
5. **Settings:** Rows link to `#` as placeholders. **Mindful eating guides** card: editable **sodium (mg/day)** + **added sugar (g/day)** + **Reset to defaults** (PRD FR-B5).
6. **Onboarding step 1:** **Indonesia only**—**city/regency** [`Select`](onboarding.html) (sample list; full data in eng). Hidden `country=ID`.
7. **Today meal ideas:** **C18** subline + **Browse all safe recipes**—generic browse path when no logs (PO).
8. **Fridge empty search:** Inline error appears when the ingredient field is cleared and **Find recipes** is clicked; default field is pre-filled for demo flow.
9. **GitHub Pages:** The monorepo’s HTML is **not** at the repo root. A copy is synced into **`docs/`** (run `./scripts/sync-prototype-to-docs.sh` from repo root). On GitHub: **Settings → Pages → Deploy from a branch → `main` → folder `/docs`** (not `/ root`). See repo root **`GITHUB-PAGES.md`**. Source of truth for edits remains this folder (`m1-hifi-prototype/`); re-sync after changes.
10. **§5.16–5.18 screens:** [`public-recipe.html`](public-recipe.html) is the **print/share-friendly** recipe view (`?print=1` triggers print). **Recipe detail → Print or PDF** opens that page. [`meal-prep.html`](meal-prep.html) is the **Prep** tab (week plan). [`growth.html`](growth.html) is linked from **Settings → Growth** (weight/height mock + disclaimer). **Bottom nav** is **five tabs** (Today, Log, Fridge, Recipes, Prep) on all authenticated shell pages that include the tab bar.
